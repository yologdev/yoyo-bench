## Script Overview

Usage:
```bash
python3 convert_masks.py <weights_path> <output_path> <rgb_path> <csv_path>
```

Algorithm:

1. Load the RGB image and metadata CSV containing cell masks (rectangles + polylines)

2. **Initialize MobileSAM** (vit_t - the TinyViT distilled variant) using the provided weights, running on CPU

3. **Encode the image once** with predictor.set_image() — MobileSAM computes the image embedding a single time for efficiency

4. **For each cell**, run MobileSAM using:
   • A box prompt from the mask's bounding box (xmin, ymin, xmax, ymax)
   • A **center point prompt** to help focus on the cell
   • `multimask_output=True` → picks the highest-scoring mask

5. Resolve overlaps using a nearest-centroid rule: pixels claimed by multiple masks are assigned to the mask whose centroid is closest

6. **Convert each binary mask to a polyline** using cv2.findContours → keep largest contour → cv2.approxPolyDP for simplification → close the polygon

7. Recompute xmin/ymin/xmax/ymax from the new polyline coordinates

8. **Save** the updated CSV to output_path/ with all masks typed as "polyline"

Result on demo data: 48/48 masks converted to polylines, overlap resolved, all validation checks pass.
