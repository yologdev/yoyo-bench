## Optimization Summary

Original query problem: The original query used two correlated subqueries in the SELECT clause to compute top_synsetid and `top_synset_sense_count. The top_synset_sense_count` subquery contained a nested correlated subquery — for each of the ~500 filtered rows, SQLite had to:
1. Scan all senses for that word to find the top synsetid
2. Then scan all senses again to count senses in that synsetid

Since there are no indexes, these were full table scans repeated thousands of times → timeout on the read-only DB.

**Optimized approach (CTEs + Window Function):**
1. `synset_counts` CTE: Aggregate senses once to get (wordid, synsetid, count) — eliminates the join to synsets for counting
2. `word_stats` CTE: Compute per-word totals with a single pass over synset_counts
3. `top_synset` CTE: Use ROW_NUMBER() window function to rank synsets per word in one pass — eliminates all correlated subqueries
4. Final join assembles the result

Result: ~3 seconds vs timeout (>30s) on the read-only database — **10x+ speedup** with identical output.
