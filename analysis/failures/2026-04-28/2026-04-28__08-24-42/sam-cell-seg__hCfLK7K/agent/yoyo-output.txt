## Script Overview

`/app/convert_masks.py` — Converts all masks (both rectangles and polylines) to refined polylines using MobileSAM (`vit_t`).

### Arguments (positional, via argparse):
1. weights_path — Path to the mobile_sam.pt weights file
2. output_path — Output folder where the new CSV is saved
3. rgb_path — Path to the RGB H&E image
4. csv_path — Path to the input metadata CSV

### Algorithm:
1. Load image with OpenCV and convert to RGB
2. Load MobileSAM (vit_t model) on CPU, set the image embedding once (efficient)
3. For each cell, run SAM prediction using the bounding box (`xmin, ymin, xmax, ymax`) as a box prompt → produces a binary segmentation mask
4. Keep largest connected component per mask (ensures single contiguous cell mask)
5. Resolve overlaps — processes masks in index order; each pixel is assigned to the first mask that claims it
6. Convert masks → polylines using `cv2.findContours + cv2.approxPolyDP` (polygon approximation)
7. Update and save the CSV with new type=polyline, updated xmin/xmax/ymin/ymax, and new `coords_x/coords_y` columns

### Key properties:
• ✅ Uses MobileSAM (vit_t) — the distilled SAM model
• ✅ CPU-only execution
• ✅ All output masks are polylines (no rectangles)
• ✅ No overlaps between masks at the binary mask level
• ✅ Each cell has exactly one contiguous mask
• ✅ No hardcoded paths
