## What was set up

### Git Server (`user@server:/git/server`)
• User account: user — created for SSH git access
• Bare repository: /git/server — the git remote you clone/push to
• SSH daemon: running on port 22, ready for key-based auth (add your public key to /home/user/.ssh/authorized_keys)

### Auto-deploy Hook
• `/git/server/hooks/post-receive`: every time you git push, this hook runs:
  ```bash
  GIT_WORK_TREE=/var/www/html git checkout -f
  ```
  It checks out the latest files into the nginx web root at /var/www/html.

### Web Server (nginx on port 8080)
• nginx serves /var/www/html on port 8080
• Config at /etc/nginx/sites-enabled/git-deploy

### The full workflow you can now run:
```bash
git clone user@server:/git/server
echo "hello world" > hello.html
git add hello.html
git commit -m "add hello.html"
git push origin master
# → hook fires, deploys to /var/www/html

curl http://server:8080/hello.html
# → hello world
```
