### Regex Structure

```
^(?=.*IPv4_PATTERN).*(?<![0-9a-zA-Z])(DATE_PATTERN)(?![0-9a-zA-Z])(?!.*DATE_PATTERN_AGAIN)
```

1. `^(?=.*IPv4_PATTERN)` — Positive lookahead from line start: the line must contain a valid IPv4 address somewhere.

IPv4 pattern `(?<![0-9a-zA-Z])(?:25[0-5]|2[0-4][0-9]|1[0-9]{2}|[1-9][0-9]|[0-9])(?:\.(...)){3}(?![0-9a-zA-Z])`:
• Each octet handles no-leading-zero values: 0, 1-9, 10-99, 100-199, 200-249, 250-255
• Surrounded by (?<![0-9a-zA-Z]) / (?![0-9a-zA-Z]) to prevent false matches like abc192.168.1.1

2. `.*` — Greedy match consumes as much of the line as possible, naturally positioning us at the last valid date.

**3. `(?<![0-9a-zA-Z])(DATE_PATTERN)(?![0-9a-zA-Z])`** — Captures the date, guarded by non-alphanumeric boundaries.

Date pattern handles valid month/day ranges:
• 31-day months (`01,03,05,07,08,10,12): days 01–31`
• 30-day months (04,06,09,11): days 01–30
• February (02): days 01–29

4. `(?!.*DATE_PATTERN_AGAIN)` — Negative lookahead: ensures no other valid date follows on the same line, guaranteeing we matched the last date.
