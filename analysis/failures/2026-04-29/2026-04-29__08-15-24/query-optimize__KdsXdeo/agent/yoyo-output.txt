### Key Optimizations

1. **Eliminated correlated subqueries** — The original query ran two correlated subqueries per output row (for `top_synsetid and top_synset_sense_count), each doing a full scan of the senses` table (212K rows). With 500+ qualifying words, that was ~1,000 full table scans, making it essentially infinite without indexes.

2. Used CTEs to materialize intermediate results —
   • word_stats CTE: computes all per-word aggregates once (same as the original subquery), filtered by the HAVING conditions.
   • synset_counts CTE: for qualifying words only (joined with word_stats), computes per-(word, synset) counts once and uses ROW_NUMBER() as a window function to rank them.

3. Window function `ROW_NUMBER()` instead of nested subqueries — Instead of re-querying senses twice per word to find the top synset, `ROW_NUMBER() OVER (PARTITION BY s.wordid ORDER BY COUNT(*) DESC, s.synsetid ASC)` computes the ranking in a single pass.

4. **Early filtering** — The synset_counts CTE joins with word_stats so only rows for qualifying words are processed (not all 212K senses rows indiscriminately).
