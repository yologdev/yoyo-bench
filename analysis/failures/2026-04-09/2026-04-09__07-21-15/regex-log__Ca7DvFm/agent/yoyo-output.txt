## Regex Structure

```
(?m)^
(?=.*  ← lookahead: line must contain a valid IPv4
  (?<![a-zA-Z0-9])            ← IP not preceded by alphanumeric
  (?:25[0-5]|2[0-4]\d|1\d{2}|[1-9]\d|\d)   ← octet (no leading zeros)
  (?:\.(?:25[0-5]|2[0-4]\d|1\d{2}|[1-9]\d|\d)){3}  ← 3 more octets
  (?![a-zA-Z0-9])             ← IP not followed by alphanumeric
)
.*?                           ← lazily advance through line
(?<![a-zA-Z0-9])             ← date not preceded by alphanumeric
(\d{4}-(?:                   ← CAPTURED date group: YYYY-
  (?:0[13578]|1[02])-(?:0[1-9]|[12]\d|3[01])  ← 31-day months
  |(?:0[469]|11)-(?:0[1-9]|[12]\d|30)          ← 30-day months
  |02-(?:0[1-9]|[12]\d)                         ← Feb 01–29
))
(?![a-zA-Z0-9])              ← date not followed by alphanumeric
(?!.*(?<![a-zA-Z0-9])DATE(?![a-zA-Z0-9]))  ← no more dates after → ensures LAST date
```

Key design decisions:
• IPv4 octet uses ordered alternation (25[0-5]|2[0-4]\d|1\d{2}|[1-9]\d|\d) to correctly reject leading zeros
• Last date is achieved by the lazy .*? combined with a negative lookahead asserting no valid date follows the matched one
• Boundary checks `(?<![a-zA-Z0-9])...(?![a-zA-Z0-9]) prevent false positives like user1134-12-1234`
