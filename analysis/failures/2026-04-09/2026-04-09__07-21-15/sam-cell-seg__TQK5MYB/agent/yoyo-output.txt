## Script Overview

**`/app/convert_masks.py`** — converts all cell masks to polylines using MobileSAM:

### Arguments (positional, via argparse)
| Argument | Description |
|----------|-------------|
| weights_path | Path to the mobile_sam.pt weights file |
| output_path | Output folder (created if needed) |
| rgb_path | Path to the RGB H&E image |
| csv_path | Path to the input metadata CSV |

### How it works

1. Loads the RGB image and CSV, initialises MobileSAM (`vit_t`) on CPU (or GPU if available)

2. **Computes image embedding once** with predictor.set_image() — efficient for processing all cells

3. For each cell, runs MobileSAM with two prompts:
   • A bounding box (from xmin/ymin/xmax/ymax)
   • A **centre-point foreground hint** (improves accuracy on small cells)
   • Picks the highest-confidence mask from the 3 returned candidates

4. Resolves overlaps using an ownership canvas:
   • Assigns pixels to cells in order of ascending mask area (smaller cells win contested pixels)
   • Strictly enforces: each pixel belongs to at most one cell

5. **Extracts contours** from the de-overlapped binary masks using cv2.findContours (largest contour only → guarantees single contiguous region per cell)

6. Outputs a CSV matching the input schema with updated `type=polyline, coords_x, coords_y, xmin, xmax, ymin, ymax`
