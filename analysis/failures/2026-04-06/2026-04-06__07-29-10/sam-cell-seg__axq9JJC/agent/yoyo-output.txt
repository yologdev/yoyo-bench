## Summary

Script: /app/convert_masks.py

**Approach**:
1. Loads MobileSAM (vit_t — the TinyViT-based distilled model) with the provided weights, running on CPU.
2. **Sets the image** once in the `SamPredictor` (efficient — encoding done once, not per cell).
3. **For each cell mask** in the CSV:
   • Uses the bounding box (`xmin, ymin, xmax, ymax`) as a SAM box prompt
   • Uses the center point of the bounding box as an additional foreground point prompt
   • Gets 3 mask candidates (`multimask_output=True`) and picks the one with the highest IoU score
   • Keeps only the **largest connected component** (ensures single contiguous mask per cell)
4. Resolves overlaps: After all masks are generated, subtracts all other masks from each mask, then re-applies largest-component selection.
5. Converts to polylines: Uses `cv2.findContours` to extract the outer contour of each binary mask, selecting the largest contour and closing it by repeating the first point.
6. Saves output: Updates `xmin, xmax, ymin, ymax, coords_x, coords_y, type columns and saves to {output_path}/{input_csv_filename}`.
