The issue is that all_gather breaks the autograd graph. The gathered tensors don't have grad_fn. I need to use a custom autograd function to handle the gradient properly through the collective communications. This is a key aspect of tensor parallelism — the backward pass needs to use the conjugate collectives.

For ColumnParallelLinear:
• Forward: all_gather (to concatenate outputs)
• Backward: scatter (split gradient back to each rank) — which means grad flows through the local part

For RowParallelLinear:
• Forward: all_reduce (sum partial outputs)
• Backward: identity (each rank gets the same gradient, which is correct since it contributed equally)

## Implementation Summary

**`/app/parallel_linear.py`** implements tensor parallelism for linear layers with two classes and supporting infrastructure:

### Custom Autograd Functions (gradient-preserving collectives)

`_AllGather` — differentiable all_gather:
• Forward: each rank contributes its local shard; outputs are concatenated along the last dimension → (*, out_features // world_size) → (*, out_features)
• Backward: splits the incoming gradient and returns only the local rank's slice — the inverse of gather

`_AllReduce` — differentiable all_reduce (sum):
• Forward: sums partial outputs across all ranks
• Backward: passes gradient through unchanged (identity) — correct because each rank contributed to the sum and should receive the full gradient

### `ColumnParallelLinear`
• **Weight: shards `master_weight` along dim 0** (output dimension) → (out//world_size, in_features) per rank
• Bias: sharded along output dim → (out//world_size,), zero-initialized
• Forward: `local_out = input @ weight.T + bias → all_gather → (*, out_features)`

### `RowParallelLinear`
• Weight: shards master_weight along dim 1 (input dimension) → (out_features, in//world_size) per rank
• Bias: full (out_features,) on every rank, zero-initialized; added once after all_reduce
• Forward: `partial = input_shard @ weight.T → all_reduce (sum) → + bias → (*, out_features)`
