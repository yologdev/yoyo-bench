## `/app/convert_masks.py` — What it does

### Pipeline overview

1. **Load image & MobileSAM** — The RGB image is loaded with Pillow and the `vit_t` (TinyViT) MobileSAM model is initialised from the provided weights file on CPU. `predictor.set_image()` computes the image embedding once (shared across all cells).

2. **Run MobileSAM on every cell* — For each row in the CSV (both rectangles and* polylines), the bounding box [xmin, ymin, xmax, ymax] is used as the box prompt to predictor.predict(). With multimask_output=True, the mask with the highest predicted IoU score is chosen, giving a precise binary segmentation mask regardless of whether the original annotation was a rectangle or polyline.

3. Resolve overlaps — All binary masks are summed. Any pixel claimed by more than one mask is assigned exclusively to the cell whose bounding-box centre is closest to that pixel.

4. **Extract contours → polylines** — cv2.findContours is called on each cleaned mask. The largest contour is selected (ensuring one contiguous region per cell) and its (x, y) points become the new coords_x / coords_y lists. Updated xmin/ymin/xmax/ymax are derived from the contour extremes.

5. Save CSV — The output DataFrame is identical to the input except type is set to "polyline" for all rows and the four coordinate columns are updated. It is saved to `{output_path}/{input_filename}`.
