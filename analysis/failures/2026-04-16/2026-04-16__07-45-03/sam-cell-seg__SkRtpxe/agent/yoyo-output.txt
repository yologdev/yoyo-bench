## Summary

Script: /app/convert_masks.py

### Arguments (positional, via argparse)
• weights_path — path to `mobile_sam.pt` weights
• output_path — output folder where the new CSV is saved  
• rgb_path — path to the RGB H&E image
• csv_path — path to the input metadata CSV

### How it works

1. Loads MobileSAM (vit_t model type, CPU-only) using mobile_sam.sam_model_registry and SamPredictor

2. Encodes the image once with predictor.set_image() for efficient batch inference

3. Runs SAM on every mask using the appropriate prompt strategy:
   • Rectangle masks → box prompt (xmin, ymin, xmax, ymax)
   • Polyline masks → bounding box + up to 3 interior point prompts (sampled from the filled polygon region)
   • Falls back to box-only if the polyline approach fails

4. **Resolves overlaps** — if two SAM masks overlap, each contested pixel is assigned to the mask whose centroid is nearest (nearest-centroid Voronoi assignment), ensuring zero overlap and **one contiguous region per cell**

5. Converts binary masks → polylines using cv2.findContours, keeping the largest contour per mask, and updates `xmin/ymin/xmax/ymax` accordingly

6. Saves the output CSV with the same columns as the input (minus any unnamed index column), with type = "polyline" for all rows
