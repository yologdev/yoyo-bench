Let me think about this carefully:

**ColumnParallelLinear**: splits weight by columns (output dimension)
- Weight shape: [out_features, in_features] (PyTorch convention)
- Split by columns means split along output dimension (rows of weight matrix)
- Each rank gets weight of shape [out_features // world_size, in_features]


- Bias is also sharded along the output dimension, and each rank computes a partial output before gathering results across all ranks

**RowParallelLinear**: splits weight by rows (input dimension)
- Weight shape: [out_features, in_features]
- Split by rows means split along input dimension (columns of weight matrix)
- Each rank gets weight of shape [out_features, in_features // world_size]
- Input is also split: each rank gets input[:, rank*chunk:(rank+1)*chunk]
- Each rank computes a partial output, then all_reduce to sum contributions across ranks, with bias applied after the reduction
