## Implementation Summary

### Helper autograd functions

Two torch.autograd.Function subclasses wire the distributed collectives into PyTorch's autograd graph so gradients flow correctly:

| Function | Forward | Backward |
|---|---|---|
| _AllGatherFunc | dist.all_gather → concat along last dim | slice the grad for this rank's partition |
| _AllReduceFunc | dist.all_reduce (SUM) | identity — grad passes through unchanged |

────────────────────────────────────────

### `ColumnParallelLinear`

• Weight sharding: master_weight[start:end, :] — splits the **output** (row) dimension into world_size equal slices.  
  Each rank holds shape [out_features/world_size, in_features].
• Bias sharding: zeros of shape [out_features/world_size] — one slice per rank.
• Forward: local_out = x @ weight.T + bias (shape […, out_f/ws]), then **`all_gather** → concat → shape […, out_features]`.
• Backward: grad of the gather is the rank's own slice of the upstream gradient.

────────────────────────────────────────

### `RowParallelLinear`

• Weight sharding: master_weight[:, start:end] — splits the input (column) dimension.
  Each rank holds shape `[out_features, in_features/world_size]`.
• Bias: full [out_features] zeros on every rank (no sharding).
• Forward: input is already partitioned; `local_out = x_shard @ weight.T (shape […, out_f]), then **all_reduce` (SUM)** → full output, then + bias.
• Backward: grad of the all_reduce is the identity (each rank receives the same upstream gradient).
