## Summary

The script /app/convert_masks.py has been written and tested successfully. Here's what it does:

### Usage
```bash
python3 /app/convert_masks.py <weights_path> <output_path> <rgb_path> <csv_path>
```

### Implementation Details

1. MobileSAM Integration: Loads `mobile_sam (vit_t` model type) with the provided weights. The image is encoded once (efficient), then each cell mask is refined using a bounding box prompt derived from the CSV's `xmin/ymin/xmax/ymax` columns.

2. CPU-only: Explicitly sets device="cpu" — no GPU assumed.

3. All masks → polylines: 
   • For each cell, SAM predicts the best mask given the bounding box prompt
   • The SAM mask is clipped to a buffer around the original bounding box to prevent bleeding
   • Falls back to rasterizing the original coordinates if SAM fails or returns empty

4. No overlaps: A global label map is built across all masks. Contested pixels (claimed by multiple masks) are assigned to the cell whose centroid is closest.

5. **Single contiguous mask**: cv2.connectedComponentsWithStats keeps only the largest connected component per cell.

6. **Contour extraction**: cv2.findContours + cv2.approxPolyDP converts the pixel mask back to a polyline, updating coords_x, coords_y, `xmin, ymin, xmax, ymax, and type='polyline'`.

7. Output: Saves a CSV matching the input format (same columns) to <output_path>/<basename_of_input_csv>.
