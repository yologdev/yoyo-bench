## Implementation Summary

### `train_step_pipeline_afab(model, inputs, targets, device, dtype)`

Layer Partitioning:
• Distributes num_hidden_layers roughly evenly across ranks using round-robin assignment of remainder layers
• Rank 0 gets layers `[0..k)` plus embed_tokens
• Middle ranks get intermediate layers
• Last rank gets final layers, norm, and lm_head

**AFAB Forward Phase (all microbatches first):**
1. **Rank 0**: Embeds input tokens, computes causal mask & rotary position embeddings, runs its assigned layers, sends hidden states `[batch, seq_len, hidden_size]` to rank 1
2. Middle ranks: Receives hidden states from previous rank (detached, with requires_grad=True), runs assigned layers, sends to next rank
3. Last rank: Runs assigned layers, applies norm + lm_head, computes cross-entropy loss scaled by 1/num_microbatches

AFAB Backward Phase (all microbatches after):
1. Last rank: Calls loss.backward() directly
2. **Middle/first ranks**: Receives gradient tensor from next rank, calls `output.backward(received_grad)`
3. Non-first ranks: Sends inp.grad (gradient w.r.t. input hidden states) to previous rank

Communication: Uses torch.distributed.P2POp with `batch_isend_irecv` for point-to-point communication between pipeline stages.
