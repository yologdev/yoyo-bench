Now I understand the full picture:

Root cause:
• In RELEASE mode (`NDEBUG), _Facet_Register_impl() allocates _Fac_node objects using ::operator new`, which routes through the custom heap allocator in main.cpp.
• `_Facet_Register_impl() is called lazily (when _Static_init_complete is true AND _Facets_registered` is false), which happens during main() after g_custom_heap is initialized — so the nodes get allocated from the custom heap buffer.
• At program exit, `_Fac_tidy_reg_t::~_Fac_tidy_reg_t() calls ::operator delete(nodeptr)`, but by then g_custom_heap is nullptr (already destroyed in shutdown()), so `std::free()` is called on a pointer that lives inside the custom heap buffer (not a real heap allocation) → SIGSEGV.
• In DEBUG mode, std::malloc/std::free are used consistently, so no crash.

Fix: In user_init(), force locale initialization (call _Facet_Register_impl indirectly by using `std::locale or streaming) **before** g_custom_heap` is set up. But we can't control when g_custom_heap is set — that happens in `Application::init() after user_init()` returns.

Actually, _Facet_Register_impl is called lazily on first use of locale. The first std::cout << in `init() happens **after** g_custom_heap` is set. We need to trigger locale initialization in user_init(), before g_custom_heap is set.
