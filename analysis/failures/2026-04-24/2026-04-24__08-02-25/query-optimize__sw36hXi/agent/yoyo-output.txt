The optimized query is saved at /app/sol.sql. Here's what I did to optimize it:

Key optimizations:

1. **Eliminated correlated subqueries** — The original query had two nested correlated subqueries in the SELECT clause to find the top_synsetid and top_synset_sense_count. These executed once per row, making the query O(N²). I replaced them with a CTE computed once.

2. Used `ROW_NUMBER()` window function — Instead of a self-join or nested subquery to find the "best" synset per word, I used ROW_NUMBER() OVER (PARTITION BY s.wordid ORDER BY COUNT(*) DESC, s.synsetid ASC) to rank synsets per word in a single pass, then filtered for `rn = 1`.

3. Filtered early — The word_synset_counts CTE only processes words that already pass the word_stats filter (`WHERE s.wordid IN (SELECT wordid FROM word_stats)`), reducing the working set significantly.

4. **Dropped the unnecessary words join from the inner subquery** — The original inner subquery joined words w into the aggregation only to use w.wordid and w.word in `GROUP BY. Since senses already has wordid`, the join to words is deferred to the final SELECT.
