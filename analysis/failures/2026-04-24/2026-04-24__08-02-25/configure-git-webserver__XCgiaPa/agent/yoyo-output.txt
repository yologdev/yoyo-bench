## What was set up

### 1. Unix user `user`
Created so that git clone user@server:/git/server maps to a real OS account. You can add your public SSH key to /home/user/.ssh/authorized_keys and login will work without a password.

### 2. Bare Git repository at `/git/server`
```
git init --bare /git/server
```
This is the repository the user clones from and pushes to.

### 3. `post-receive` hook — `/git/server/hooks/post-receive`
```bash
#!/bin/bash
GIT_WORK_TREE=/var/www/gitsite GIT_DIR=/git/server git checkout -f
find /var/www/gitsite -type d -exec chmod 775 {} \;
find /var/www/gitsite -type f -exec chmod 664 {} \;
echo ">>> Deployed to /var/www/gitsite"
```
Every git push automatically checks out the latest files into the web root.

### 4. nginx serving `/var/www/gitsite` on port 8080
Config at /etc/nginx/sites-available/gitsite:
```nginx
server {
    listen 8080;
    server_name _;
    root /var/www/gitsite;
    index index.html;
    location / {
        try_files $uri $uri/ =404;
    }
}
```

### Flow
```
git push  →  post-receive hook  →  git checkout -f → /var/www/gitsite  →  nginx:8080
```

To authorize your SSH key, simply append it to `/home/user/.ssh/authorized_keys` on the server.
