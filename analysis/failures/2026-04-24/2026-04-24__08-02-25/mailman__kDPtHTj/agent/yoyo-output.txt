## Summary

Here's what was set up to get the reading-group@local.edu mailing list working:

### Postfix (`/etc/postfix/main.cf`)
• Set myhostname = local.edu and myorigin = local.edu so the server identifies as local.edu
• Set `mydestination = local.edu, localhost, ... to accept local mail for local.edu` users
• Set mail_spool_directory = /var/mail to deliver mail to `/var/mail/<username>` mbox files
• Added `transport_maps = hash:/var/lib/mailman3/data/postfix_lmtp` — routes list addresses to Mailman via LMTP on port 8024
• Added `relay_domains = hash:/var/lib/mailman3/data/postfix_domains — accepts mail for the local.edu` Mailman domain
• Expanded `local_recipient_maps to include the Mailman LMTP map so list addresses (reading-group-join@local.edu`, etc.) aren't rejected as unknown local users

### Mailman3 (`/etc/mailman3/mailman.cfg`)
• Configured with `layout: debian pointing at /var/lib/mailman3`
• Set `lmtp_host: 127.0.0.1 / lmtp_port: 8024` for Postfix LMTP integration
• Uses SQLite at /var/lib/mailman3/data/mailman.db

### Mailing List
• Created domain local.edu in Mailman
• Created reading-group@local.edu with:
  • `SubscriptionPolicy.open` — no owner approval needed for joins
  • _preferred_language = 'en' — required for Mailman to process commands
• Regenerated Postfix transport maps via mailman aliases
