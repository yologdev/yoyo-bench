## Summary

I wrote `/app/convert_masks.py that uses **MobileSAM** (vit_t` — the TinyViT-based distilled model) to refine all cell masks from the input CSV and convert them all to polylines.

### How the script works:

1. Argument parsing — Takes weights_path, output_path, `rgb_path, and csv_path via argparse`.

2. Model loading — Loads MobileSAM using `sam_model_registry["vit_t"]` with the provided weights, runs on CPU (or GPU if available).

3. Image encoding — Calls `predictor.set_image()` once to encode the full image (efficient).

4. Per-mask refinement — For each cell (both rectangles and polylines):
   • Uses the bounding box (xmin/ymin/xmax/ymax) as the SAM box prompt
   • Uses the centroid as a positive point prompt
   • Runs predictor.predict() with `multimask_output=True` and picks the highest-scoring mask

5. Overlap resolution — Masks are deduplicated by giving priority to higher-scoring masks (pixels claimed by a better mask are removed from lower-scoring ones), ensuring no overlap and **one contiguous region per cell**.

6. Polyline extraction — cv2.findContours() extracts the largest contour from each binary mask, producing closed polygon coordinates.

7. Output — Saves a CSV matching the input format with updated type=polyline, `xmin/xmax/ymin/ymax, coords_x, and coords_y` columns.
