## Implementation Summary

### `ColumnParallelLinear`
• **Weight sharding: The master weight `[out_features, in_features]` is split along dim 0** (output dimension). Each rank gets [out_features // world_size, in_features].
• Bias sharding: The bias is also sharded along the output dimension — each rank holds [out_features // world_size], initialized to zero.
• Forward pass: Each rank computes a partial output [..., local_out_features], then dist.all_gather collects all shards and concatenates them along the last dimension to produce the full [..., out_features] output.

### `RowParallelLinear`
• Weight sharding: The master weight [out_features, in_features] is split along dim 1 (input dimension). Each rank gets [out_features, in_features // world_size].
• Bias: The bias is not sharded — every rank holds the full [out_features] bias, initialized to zero.
• **Forward pass**: Each rank computes a partial output [..., out_features] from its input shard, then `dist.all_reduce` (SUM) combines all partial results. The bias is added after the reduction (only once, not once per rank).

### Design choices
• Both classes detect distributed state via dist.is_available() and dist.is_initialized(), gracefully falling back to world_size=1, rank=0 for non-distributed usage.
• Weights are extracted with .contiguous().clone() to ensure each rank owns its own memory and gradients flow correctly.
• Communication uses standard PyTorch distributed primitives (all_gather, all_reduce) compatible with any backend.
