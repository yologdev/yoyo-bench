## Implementation Summary

### Key Design Decisions

Autograd-aware collectives — Standard dist.all_gather and dist.all_reduce break the autograd graph. Two custom torch.autograd.Function subclasses solve this:

• _AllGather: forward gathers tensors from all ranks and concatenates along the last dim; backward slices the upstream gradient to return only this rank's chunk.
• _AllReduce: forward clones and reduces (sum) across ranks; backward passes the gradient unchanged (all_reduce is self-adjoint for the sum operation).

### `ColumnParallelLinear`
• **Weight sharding*: `master_weight[rankchunk : (rank+1)*chunk, :]` — splits rows of the [out_features, in_features] weight matrix (i.e., output dimension columns)
• Bias sharding: `zeros(chunk_size)` — one chunk of the output bias per rank
• Forward: local F.linear → all_gather + cat along last dim → full `[batch, out_features]`

### `RowParallelLinear`
• Weight sharding: master_weight[:, rank*chunk : (rank+1)*chunk] — splits columns of the weight matrix (i.e., input dimension rows)
• Bias: full zeros(out_features) on every rank, added after the all_reduce (so it's counted exactly once)
• Forward: local `F.linear (no bias) → all_reduce sum → add bias → full [batch, out_features]`
