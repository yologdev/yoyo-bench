## Summary

I've written `/app/convert_masks.py` that converts all masks (both rectangles and polylines) to refined polylines using MobileSAM (`vit_t` — the TinyViT-based distilled SAM).

### How it works:

1. **Loads MobileSAM** (vit_t model type) with the provided weights checkpoint on CPU.

2. Computes image embeddings once using `predictor.set_image()` — efficient for processing many masks from the same image.

3. For each mask, runs SAM with two prompts:
   • A **bounding box prompt** ([xmin, ymin, xmax, ymax]) from the CSV
   • A **center point prompt** (foreground) at the bbox center
   • Picks the highest-scoring mask from the multi-mask output
   • Clips the mask to stay within the original bbox region (with a small 5px padding)
   • Keeps only the largest connected component (ensures single contiguous mask per cell)

4. Resolves overlaps between masks: any pixel claimed by multiple masks is assigned to the mask whose centroid is nearest.

5. Converts each binary mask to a polyline using cv2.findContours → CHAIN_APPROX_SIMPLE, then updates coords_x, `coords_y, xmin, xmax, ymin, ymax, and type='polyline'`.

6. **Saves** the updated CSV to output_path/ with the same filename as the input.
