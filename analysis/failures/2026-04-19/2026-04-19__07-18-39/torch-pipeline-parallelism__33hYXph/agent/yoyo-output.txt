## Summary

I implemented /app/pipeline_parallel.py with the `train_step_pipeline_afab` function for pipeline parallel training of LLaMA.

### Key Design Decisions:

1. Layer Partitioning: Transformer layers are split roughly evenly across ranks using a base + remainder strategy. With num_layers=4 and world_size=2, each rank gets 2 layers.

2. Stage Responsibilities:
   • Rank 0 (first stage): Runs embed_tokens + its transformer layers
   • Middle stages: Receives hidden states, runs transformer layers, sends to next
   • Last rank: Runs its transformer layers + norm + lm_head + computes cross-entropy loss (scaled by num_microbatches)

3. AFAB Scheduling: All forward passes complete before any backward pass starts. Hidden states from forward are kept in forward_outputs list.

4. Gradient Flow:
   • On rank 0, h_input keeps its connection to embed_tokens (has grad_fn), so backward naturally computes gradients for embed_tokens.weight
   • On other ranks, received tensors are .detach().requires_grad_(True) leaf tensors — backward computes `d(loss)/d(h_input)` which is sent back to the previous stage

5. **P2P Communication**: Uses torch.distributed.P2POp + batch_isend_irecv for point-to-point communication between adjacent pipeline stages.

Verification: Both world_size=1 and world_size=2 produce gradients with exactly zero difference compared to the reference sequential model.
