## Regex Structure

```
^(?=.*IPv4).*(?<![0-9a-zA-Z])(DATE)(?![0-9a-zA-Z])(?!.*(?<![0-9a-zA-Z])DATE(?![0-9a-zA-Z]))
```

### Components:

IPv4 Lookahead (?=.*IPv4_PATTERN):
• Each octet: (?:25[0-5]|2[0-4][0-9]|1[0-9]{2}|[1-9][0-9]|[0-9]) — covers 0–255, no leading zeros
• Surrounded by (?<![0-9a-zA-Z]) / `(?![0-9a-zA-Z]) to prevent false matches like 192.168.001.1`
• Anchored at ^ with re.MULTILINE so it checks each line

Date Pattern (captured group):
• [0-9]{4} — year
• 31-day months: (?:0[13578]|1[02])-(?:0[1-9]|[12][0-9]|3[01])
• 30-day months: (?:0[469]|11)-(?:0[1-9]|[12][0-9]|30)
• February: `02-(?:0[1-9]|[12][0-9])` — days 01–29
• Boundary assertions prevent partial matches (e.g., 1134-12-1234)

Last Date: Greedy .* before the date naturally positions at the last match; the negative lookahead (?!.*DATE) enforces no subsequent date exists on the same line.
