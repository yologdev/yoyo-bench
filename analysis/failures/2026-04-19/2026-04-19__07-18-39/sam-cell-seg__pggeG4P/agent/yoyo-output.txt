## `/app/convert_masks.py` — What it does

1. Loads MobileSAM (`vit_t` architecture) from the provided weights path and runs on CPU (with automatic GPU fallback if available).

2. For every row in the CSV (both rectangle and polyline types):
   • Builds a bounding-box prompt from the stored `xmin/xmax/ymin/ymax` columns (with a small 4-pixel pad).
   • Builds a point prompt at the centroid of the existing mask.
   • Runs predictor.predict(...) with `multimask_output=True` and picks the highest-scoring mask.

3. **Overlap resolution**: Assigns each pixel to the cell whose mask is smallest (small cells win over large neighbours), ensuring no overlapping masks.

4. **Single connected component**: Keeps only the largest connected component per mask, so each cell has exactly one contiguous mask.

5. Polyline extraction: Uses `cv2.findContours with CHAIN_APPROX_TC89_L1` to extract a compact closed polyline from each refined binary mask.

6. **Saves output CSV** with updated coords_x, coords_y, xmin, ymin, xmax, ymax and `type = 'polyline'` for all rows.
