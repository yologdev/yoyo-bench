## Summary

**`/app/convert_masks.py`** — A Python script that:

### How it works:

1. **Loads MobileSAM** (vit_t model type from `mobile_sam` package) using the provided weights path. Runs on CPU (with automatic GPU detection if available).

2. **Encodes the image once** with predictor.set_image() — the image embedding is computed once and reused for all mask predictions.

3. **For each mask in the CSV**, runs MobileSAM predict() with:
   • A bounding box prompt (xmin, ymin, xmax, ymax) — works for both rectangles and polylines
   • A **foreground center point** (centroid of the bounding box) to help SAM focus on the cell

4. Resolves overlaps greedily: pixels already claimed by an earlier mask are removed from later masks (binary mask level, ensuring strict non-overlap).

5. Converts each binary mask to a polyline:
   • Selects the largest connected component (guarantees single contiguous mask)
   • Extracts the outer contour using cv2.findContours
   • Closes the polygon by repeating the first point

6. Saves the output CSV with updated type, xmin, xmax, ymin, ymax, coords_x, coords_y columns — all masks set to "polyline".
