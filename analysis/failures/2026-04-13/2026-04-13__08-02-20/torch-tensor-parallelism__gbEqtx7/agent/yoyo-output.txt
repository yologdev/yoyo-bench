## Design Overview

### `ColumnParallelLinear`
• **Weight sharding: Splits `master_weight` along dim=0** (output dimension) → each rank gets shape (out_features // world_size, in_features)
• Bias: Sharded to match the local output size → shape (out_features // world_size,), zero-initialized
• Forward: Computes local `y_r = x @ W_r^T + b_r, then **all_gather** along the last dimension to produce the full (..., out_features)` output
• Backward (via _AllGather): Receives the full output gradient and slices only the local rank's portion back

### `RowParallelLinear`
• Weight sharding: Splits master_weight along dim=1 (input dimension) → each rank gets shape (out_features, in_features // world_size)
• Bias: Not sharded — each rank holds the full `(out_features,)` bias, zero-initialized; added after the all_reduce
• Forward: Computes local y_r = x_r @ W_r^T, then all_reduce (sum) to accumulate partial outputs, then adds full bias
• Backward (via _AllReduce): Gradient passes through unchanged (identity)

### Supporting Infrastructure
• _get_world_size() / _get_rank(): Safe wrappers that fall back to (1, 0) when dist is not initialized
• _AllGather / _AllReduce: Custom torch.autograd.Function subclasses with correct forward (collective ops) and backward (gradient routing) semantics
