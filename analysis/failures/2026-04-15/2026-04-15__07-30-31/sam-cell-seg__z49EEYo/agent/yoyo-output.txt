## How it works

Architecture: Uses MobileSAM (vit_t — the TinyViT distilled model) via sam_model_registry and SamPredictor.

Pipeline:

1. Load inputs — reads the RGB image and metadata CSV (with `xmin/xmax/ymin/ymax/coords_x/coords_y` columns).

2. Single embedding pass — calls predictor.set_image() once, so the expensive ViT encoding happens only once for the whole image (efficient on CPU).

3. Per-mask prediction — for each row, constructs two SAM prompts:
   • A bounding-box prompt from `[xmin, ymin, xmax, ymax]`
   • A foreground point at the centre of that box
   
   SAM returns 3 candidate masks; the one with the highest predicted IoU score is selected. The mask is clipped to the bounding box region.

4. Overlap removal — a pixel-level canvas assigns each pixel to the first mask that claims it, guaranteeing zero overlap between cells.

5. **Contour extraction** — cv2.findContours with `CHAIN_APPROX_TC89_L1` extracts the outer boundary of each mask as a closed polyline, selecting the largest contour to ensure one contiguous region per cell.

6. Save — writes the output CSV to output_path/ with updated `xmin/xmax/ymin/ymax/coords_x/coords_y columns and type = "polyline"` for every row.
