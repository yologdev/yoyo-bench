## Summary

### Problem Discovery
The /app/main.db was in WAL (Write-Ahead Logging) mode but the WAL file was missing. Through investigation, I found that SQLite installed in this container was a modified build — its version string ends with ccalt1 instead of the standard hash, indicating an alternative checksum algorithm.

### Root Cause
The WAL file's corruption was caused by using a modified checksum algorithm: instead of reading 32-bit words in big-endian format (as standard SQLite does when magic = 0x377f0682), this custom build reads them in little-endian format. This meant any WAL file created by this SQLite couldn't be read by standard SQLite verification, and vice versa.
