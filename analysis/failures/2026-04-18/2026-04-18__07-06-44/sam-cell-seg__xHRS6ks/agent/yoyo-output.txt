## Summary

The /app/convert_masks.py script:

### Arguments (via argparse)
• weights_path — path to the MobileSAM .pt weights file
• output_path — output folder where the new CSV will be saved
• rgb_path — path to the RGB H&E image
• csv_path — path to the input metadata CSV

### Algorithm

1. Load the RGB image and CSV metadata
2. Load MobileSAM (`vit_t` — the TinyViT-based distilled model) using SamPredictor on CPU
3. Set the image once (efficient — embedding is computed once for all masks)
4. For each mask, use the bounding box (xmin, ymin, xmax, ymax) as a SAM prompt → get the highest-scoring predicted mask
5. Resolve overlaps: process masks in order, removing pixels already claimed by earlier masks; keep only the largest connected component per mask (ensuring one contiguous region per cell)
6. Convert to polyline: extract contours with cv2.findContours, taking the largest contour, closing it by appending the first point, and updating `coords_x, coords_y, xmin, xmax, ymin, ymax`
7. Save the updated CSV (same columns as input, just updated geometry + type = "polyline" for all rows)

### Key design decisions
• Uses vit_t (MobileSAM's TinyViT encoder), not the original heavy SAM
• Runs entirely on CPU (device="cpu")
• No overlap between masks; each cell gets exactly one contiguous mask
• Robust fallbacks: if SAM fails or the cleaned mask is empty, falls back to bounding-box geometry
