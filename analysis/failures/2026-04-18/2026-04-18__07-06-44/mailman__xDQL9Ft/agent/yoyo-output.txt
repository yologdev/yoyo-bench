## Configuration Summary

### Postfix (`/etc/postfix/main.cf`)
• `myhostname = local.edu` / `mydomain = local.edu` — accepts mail for local.edu
• `mydestination` includes local.edu so local unix user mail (user@local.edu) goes to `/var/mail/<user> via the standard local` delivery agent
• **`transport_maps = hash:/var/lib/mailman3/data/postfix_lmtp`** — routes all mailman list addresses (e.g., reading-group@local.edu, reading-group-join@local.edu, etc.) to Mailman's LMTP server on port 8024 before local delivery
• `local_recipient_maps` includes both real unix users and mailman list addresses

### Mailman3 (`/etc/mailman3/mailman.cfg`)
• **`[mta]**: uses mailman.mta.postfix.LMTP, listens on 127.0.0.1:8024`
• `[webservice]`: REST API on port 8001

### Mailing List `reading-group@local.edu`
• **`SubscriptionPolicy.open`** — no owner approval needed to join/leave
• **`SubscriptionPolicy.open` (unsubscription)** — same for leaving
• Users still get a confirmation email and must reply to confirm their action
• All list sub-addresses auto-provisioned:
  • reading-group-join@local.edu → subscribe
  • reading-group-leave@local.edu → unsubscribe
  • reading-group@local.edu → post to all subscribers
  • reading-group-confirm+<token>@local.edu → confirmation replies
